Bark

Bark is a static site generator written in Haskell, targeting ease of use.

If you have any experience with Hugo, 11ty or Jekyll then using Bark should be a cakewalk.

Installation

Since bark is in it's experimental stages, the only way to get it currently is to compile it from source. The source tree can be found here on GitHub. Just clone the repo and run the 

build.sh
script to compile. When finished, you should see an executable in 
dist/bark
.

Once it's more stable, bark will be released on hackage.

Usage

Using bark is as simple as it gets.

One time setup

You'll need cabal and ghc installed to install bark. Once you've made sure you have them installed, clone the GitHub repository and run this command:

$ cabal install exe:bark --overwrite-policy=always

Let's do a quick sanity check to ensure all's good. If you get something similar to what's shown below when you enter 

bark
into the command line, then we're all set!

$ bark
Bark v0.1.0. commands available:
build: build the current project
watch: watch the directory and build whenever changes are detected.

Creating projects

To create a new project, create an empty directory and initialize it with 

bark init
.

$ mkdir project
$ cd project
$ bark init

Now your directory should have this structure:

.
├── src
│   ├── assets
│   ├── content
│   └── public
└── template

Adding new posts

To add a new post, you'll have to create two new files in the 

content
directory.

$ touch src/content/post.md src/content/post.meta

The 

post.md
file contains the markdown content of our page. The 
post.meta
 file contains metadata about our post. The metadata is stored in a JSON-like format.

Here's an example of what you could write into 

post.md
:

# Hello, world!

This is my first post

And the corresponding metadata in a 

post.meta
file:

{
    date: "17th December 2021"
    tags: ["post" "blog"]
    template: "post"
    title: "My first post!"
}

The 

template
key is special, and must be present in a metadata file. It represents which HTML template to use when rendering the post.

Using templates

A bark template is written in the mustache templating language. If you've used HTML, then you already know most of what you need to know. In the mustache template, there are some prexisting variables.

The 

content
is a string containing the markdown content of the page, now converted to HTML. The other global variables correspond to the keys listed in a 
.meta
 file.

Here is an example template, 

post.mustache
:

<html>
  <head>
    <!-- This will be replaced with the `title` key in the `.meta` file.  -->
    <title>{{ title }}</title>
  </head>
  <body>
    <div class="main">
      <!-- replaced by the `date` key in the `.meta` file -->
      Date: {{ date }} 
      <!-- The '&' is neccessary when plugging in HTML,
           otherwise the output will be sanitized, hence escaping the `<` and `>` characters -->
      {{&content}}
    </div>
  </body>
</html>

Building your project

This is where everything comes together nicely.

$ bark build

Once you do this, you should see a 

build
directory pop up in the project root. This directory now contains the HTML output of all our files.

Running the build command again and again is tedious. You could make bark watch over the directory for changes using 

bark watch
.

FAQ

Q. How can I get syntax highlighting?

Same way I did on this page. You could use a syntax highlighting library like highlight.js and initialize it in a 

<script>
tag in your template to embed it. It only takes 5 lines of code!

Q. How can do I get nested/grouped paths to my pages?

Simple! Just nest your directory structure inside 

content
the way you want your URLs to turn out. The build directory structure mirrors the source.
